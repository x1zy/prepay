# 📦 Деплой на GitHub Pages - Пошаговая инструкция

## 🚀 Быстрый старт

### Шаг 1: Включите GitHub Pages в настройках репозитория

1. Откройте ваш репозиторий на GitHub
2. Перейдите в **Settings** (Настройки)
3. В левом меню найдите **Pages**
4. В разделе **Source** выберите **GitHub Actions**
5. Сохраните изменения

### Шаг 2: Настройте Secrets (если нужен бэкенд API)

Если ваше приложение использует внешний API, добавьте переменные окружения:

1. В репозитории перейдите в **Settings** → **Secrets and variables** → **Actions**
2. Нажмите **New repository secret**
3. Добавьте следующие secrets (если нужны):
   - **Name:** `VITE_BICYCLE_API_URL`
     **Value:** URL вашего API бэкенда (например: `https://api.yourdomain.com`)
   
   - **Name:** `VITE_BICYCLE_API_KEY`
     **Value:** Ваш API ключ (если требуется)

> ⚠️ **Важно:** Если у вас нет бэкенда, secrets можно не добавлять. Приложение будет работать, но API запросы не будут выполняться.

### Шаг 3: Задеплойте приложение

#### Вариант A: Автоматический деплой (рекомендуется)

1. Сделайте commit и push в ветку `main` или `master`:
   ```bash
   git add .
   git commit -m "Deploy to GitHub Pages"
   git push origin main
   ```

2. GitHub Actions автоматически:
   - Соберет приложение
   - Задеплоит на GitHub Pages

3. Проверьте статус деплоя:
   - Перейдите в **Actions** в вашем репозитории
   - Дождитесь завершения workflow "Deploy to GitHub Pages"
   - Если все успешно, приложение будет доступно по адресу: `https://x1zy.github.io/prepay/`

#### Вариант B: Ручной деплой через GitHub Actions

1. Перейдите в **Actions** в вашем репозитории
2. Выберите workflow **Deploy to GitHub Pages**
3. Нажмите **Run workflow**
4. Выберите ветку (обычно `main`)
5. Нажмите **Run workflow**

#### Вариант C: Ручной деплой через npm (альтернатива)

```bash
# Установите зависимости (если еще не установлены)
npm install

# Соберите приложение
npm run build

# Задеплойте на GitHub Pages
npm run deploy
```

> ⚠️ **Примечание:** Этот метод использует `gh-pages` пакет и требует настройки GitHub token.

### Шаг 4: Проверьте результат

После успешного деплоя:

1. Откройте `https://x1zy.github.io/prepay/` в браузере
2. Проверьте консоль браузера (F12) на наличие ошибок
3. Убедитесь, что все функции работают корректно

## 🔧 Решение проблем

### Проблема: GitHub Pages показывает 404

**Решение:**
- Убедитесь, что в `vite.config.ts` установлен правильный `base: '/prepay/'`
- Проверьте, что workflow завершился успешно
- Подождите несколько минут - GitHub Pages может обновляться с задержкой

### Проблема: Workflow не запускается

**Решение:**
- Убедитесь, что файл `.github/workflows/deploy.yml` существует
- Проверьте, что вы делаете push в ветку `main` или `master`
- Убедитесь, что GitHub Pages включен в настройках

### Проблема: Ошибки при сборке

**Решение:**
- Проверьте логи в **Actions** → выберите failed workflow → посмотрите ошибки
- Убедитесь, что все зависимости установлены (`npm ci`)
- Проверьте, что TypeScript компилируется без ошибок (`npm run build` локально)

### Проблема: API запросы не работают

**Решение:**
- Убедитесь, что `VITE_BICYCLE_API_URL` правильно настроен в Secrets
- Проверьте, что бэкенд доступен по указанному URL
- Проверьте CORS настройки на бэкенде

## 📝 Структура деплоя

```
Ваш репозиторий
├── .github/
│   └── workflows/
│       └── deploy.yml          # Автоматический деплой
├── src/                        # Исходный код
├── public/                     # Статические файлы
├── dist/                       # Собранное приложение (создается при build)
├── vite.config.ts             # Конфигурация Vite (base: '/prepay/')
└── package.json                # Зависимости и скрипты
```

## 🎯 Что происходит при деплое

1. **GitHub Actions запускается** при push в `main`/`master`
2. **Устанавливаются зависимости** (`npm ci`)
3. **Собирается приложение** (`npm run build`) с переменными окружения из Secrets
4. **Создается папка `dist/`** с готовым приложением
5. **Деплоится на GitHub Pages** через официальный action
